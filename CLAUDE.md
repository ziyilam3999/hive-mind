# Hive Mind v3 — AI Agent Instructions

## Project Overview
Hive Mind is an AI orchestrator that decomposes a PRD into stories and executes them via Claude subagents.
TypeScript ESM project. npm-publishable CLI tool.

## File Structure Rules
- **Source code:** `src/` — all `.ts` files organized by concern
  - `src/stages/` — pipeline stage implementations
  - `src/agents/` — spawner, prompts, model-map, tool-permissions
  - `src/reports/` — parser, templates
  - `src/state/` — execution-plan, checkpoint, manager-log
  - `src/types/` — shared type definitions
  - `src/utils/` — file-io, shell, token-count, backoff
  - `src/config/` — schema, loader
  - `src/memory/` — manager, graduation
  - `src/manifest/` — generator
  - `src/tooling/` — detect, setup
- **Tests:** `src/__tests__/` — vitest, mirrors source structure
- **Build output:** `dist/` — .gitignored, generated by `npm run build`
- **Design repo:** `../hive-mind-design/` — PRD, SPEC, plans, e2e results
- **Knowledge repo:** `../.hive-mind-persist/` — symlink to `ai-brain/hive-mind-persist/` (memory, knowledge-base, constitution)

## Document Lifecycle
- **Exploration/research docs** → design repo `../hive-mind-design/design/`
- **Execution reports** → design repo `../hive-mind-design/plans/reports/`
- **Graduated learnings** → `../.hive-mind-persist/knowledge-base/` + `../.hive-mind-persist/memory.md`
- See design repo CLAUDE.md for the full decision tree.

## Commands
- `npm run build` — compile TypeScript
- `npm run test` — run vitest suite
- `npm run lint` — run eslint
- `npx tsc --noEmit` — type-check only

## Conventions
- ESM only — no `require()`, use `import`
- Mock `agents/spawner` and `utils/shell` in tests — never spawn real Claude in unit tests
- Config threading via parameters, not global singletons (P32)
- vi.mock with inline factory to avoid TDZ (P33)
- Strict output contract — no fallback writes (P34)

## Shell Safety
- Never pipe long-running process output through `head`, `tail`, or other pipe-closing commands — causes SIGPIPE
- Redirect to a log file first, then read the log: `command > /tmp/output.log 2>&1`

## Test Hygiene
- Before writing/updating tests, verify mock signatures match current source function signatures
- Check for renamed types/exports via Grep before importing in test files
- Run `npx tsc --noEmit` after writing tests to catch mismatches before `npm test`

## Agent Prompt Architecture (Hybrid Model)
- Agent identity (ROLE, RULES, few-shot examples) lives in `.claude/skills/` SKILL.md files
- Dynamic context (INPUT, OUTPUT, MEMORY, CONSTITUTION, ROLE REPORTS) stays in `buildPrompt()` in `src/agents/prompts.ts`
- `AGENT_REGISTRY` in `src/agents/registry.ts` is the single source of truth for agent metadata
- Skills are generated from registry via `src/skills/builder.ts` (not hand-maintained)
- Pipeline injects skills into target project via `src/skills/installer.ts` at pipeline start
- ~21 Tier 1 agents convert to skills; ~15 Tier 3 agents (empty rules, JSON output) stay as code
- See `docs/harness-roadmap/roadmap-releases.md` Pre-R2 + R2 for implementation timeline
