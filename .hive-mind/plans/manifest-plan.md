# ENH-15: AI-First Manifest File (Live Document)

## Context

When an AI agent starts a new session with the Hive Mind codebase, it must explore ~10 files to orient itself (README, BACKLOG, types, orchestrator, stages, etc.). This wastes tokens and time. There's no single-file entry point that gives an agent everything it needs to navigate efficiently.

The manifest is a **live document** — its dynamic sections auto-regenerate at pipeline stage boundaries.

---

## Design

### Two-Part Structure

| Part | Content | Update mechanism |
|------|---------|-----------------|
| **Static** | Project identity, architecture, source map, navigation hints, conventions | Manual edits only (~500 tokens) |
| **Dynamic** | Artifact inventory — files in `.hive-mind/` with sizes and timestamps | Auto-generated by `updateManifest()` |

### How Changes Are Tracked

**All `.hive-mind/` file I/O flows through a small set of functions:**

| Function | Location | Coverage |
|----------|----------|----------|
| `writeFileAtomic()` | `src/utils/file-io.ts` | ~95% of writes (spec, plans, reports, memory, checkpoint, execution-plan) |
| `appendFileSync()` | `src/state/manager-log.ts`, `src/memory/graduation.ts` | Log entries, KB graduation |
| `copyFileSync()` | `src/stages/execute-verify.ts` | Report archiving (test-report-N, eval-report-N) |
| `ensureDir()` | `src/utils/file-io.ts` | Directory creation |

### How Updates Are Triggered

**Stage-boundary hooks in `orchestrator.ts`** — after each stage completes, call `updateManifest()`:

```
runSpecStage()    → updateManifest()
runPlanStage()    → updateManifest()
runExecuteStage() → updateManifest()  (also after each story completes)
runReportStage()  → updateManifest()
```

**Why stage boundaries, not per-write:**
- `writeFileAtomic()` fires dozens of times per run (execution-plan.json alone updates on every story status change and attempt increment)
- Stage boundaries are natural checkpoints where the artifact set is stable
- Only 4-6 manifest updates per full pipeline run — low overhead

**Manual trigger:** `hive-mind manifest` CLI command for refreshes outside pipeline runs (e.g., after manually adding a plan or review file).

## Step 1: Create `src/manifest/generator.ts`

**Exports:**

```typescript
export async function updateManifest(hiveMindDir: string): Promise<void>
```

**Behavior:**

1. Read existing `MANIFEST.md` if it exists — extract everything above the `## Artifact Inventory` marker (this is the static content)
2. If no existing manifest, use the built-in static template (see Step 2)
3. Scan `.hive-mind/` recursively with `readdirSync` + `statSync`
4. Group files by category:
   - `spec/` → Spec artifacts
   - `plans/` → Plan artifacts (including `role-reports/`, `steps/`)
   - `reports/` → Per-story report artifacts
   - `knowledge-base/` → Graduated KB entries
   - Root state files → `.checkpoint`, `manager-log.jsonl`, `memory.md`, `consolidated-report.md`, `retrospective.md`
5. Format as markdown tree with file size (KB) and last-modified date
6. Append `> Auto-generated at {ISO timestamp} — do not edit below this line` header
7. Write via `writeFileAtomic()`

**Exclusions:** Skip `MANIFEST.md` itself, `.tmp-*` files (atomic write temporaries).

## Step 3: Hook into `orchestrator.ts`

Import `updateManifest` and call after each stage:

```typescript
import { updateManifest } from "./manifest/generator.js";

// After each stage:
await runSpecStage(hiveMindDir, prdPath, feedbackMemory);
await updateManifest(hiveMindDir);

await runPlanStage(hiveMindDir, specPath, feedbackMemory);
await updateManifest(hiveMindDir);

// In the story execution loop, after each story completes:
await updateManifest(hiveMindDir);

await runReportStage(hiveMindDir, plan);
await updateManifest(hiveMindDir);
```

Wrapped in try/catch — manifest update failures are logged but non-blocking (never crash the pipeline for a manifest update).

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Manifest update slows pipeline | Filesystem scan is lightweight (<50ms for typical `.hive-mind/` size). Non-blocking try/catch. |
| Static sections accidentally overwritten | Generator preserves everything above `## Artifact Inventory` marker. Clear "do not edit below" warning. |
| Manifest drifts from reality between pipeline runs | `hive-mind manifest` CLI command for manual refresh. CLAUDE.md pointer reminds agents to check it. |
| Large `.hive-mind/` directory with many story reports | Group by category, show file count per story instead of listing every file if >20 files in a story dir. |
