# Hive Mind v3 — AI Agent Instructions

## Project Overview
Hive Mind is an AI orchestrator that decomposes a PRD into stories and executes them via Claude subagents.
TypeScript ESM project. npm-publishable CLI tool.

## File Structure Rules
- **Source code:** `src/` — all `.ts` files organized by concern
  - `src/stages/` — pipeline stage implementations
  - `src/agents/` — spawner, prompts, model-map, tool-permissions
  - `src/reports/` — parser, templates
  - `src/state/` — execution-plan, checkpoint, manager-log
  - `src/types/` — shared type definitions
  - `src/utils/` — file-io, shell, token-count, backoff
  - `src/config/` — schema, loader
  - `src/memory/` — manager, graduation
  - `src/manifest/` — generator
  - `src/tooling/` — detect, setup
- **Tests:** `src/__tests__/` — vitest, mirrors source structure
- **E2E test PRDs:** `e2e/prds/` — reusable smoke test input PRDs
- **E2E results:** `e2e/results/` — .gitignored, local only
- **Docs:** `docs/` — BACKLOG.md, framework-comparison.md
- **Build output:** `dist/` — .gitignored, generated by `npm run build`

## Smoke Test Workflow
1. PRD templates live in `e2e/prds/` (version-controlled, reusable)
2. Execution happens in `$TEMP/hm-e2e-XXXX/` (ephemeral)
3. Results archived selectively to design repo `.hive-mind/e2e-smoke-test/run-XX/`
4. Never commit generated smoke test artifacts (src/, reports/, spec/) to this repo

## Document Lifecycle
- **Exploration/research docs** → design repo `.hive-mind/design/`
- **Implementation plans** (with file paths + ACs) → `.hive-mind/plans/` in THIS repo
- **Execution reports** → `.hive-mind/reports/US-XX/`
- **Graduated learnings** → design repo `.hive-mind/knowledge-base/` + `memory.md`
- See design repo CLAUDE.md for the full decision tree.

## Commands
- `npm run build` — compile TypeScript
- `npm run test` — run vitest suite
- `npm run lint` — run eslint
- `npx tsc --noEmit` — type-check only

## Conventions
- ESM only — no `require()`, use `import`
- Mock `agents/spawner` and `utils/shell` in tests — never spawn real Claude in unit tests
- Config threading via parameters, not global singletons (P32)
- vi.mock with inline factory to avoid TDZ (P33)
- Strict output contract — no fallback writes (P34)
